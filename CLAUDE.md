## Project Overview

PHIVE (Philip Helger Integrative Validation Engine) is a generic business document validation engine. It validates XML documents through a layered "Validation Pyramid" approach using XML Schema (XSD) and Schematron rules. Originally built for Peppol, it now supports many document types.

## Build Commands

```bash
mvn install                    # Build and install all modules
mvn test                       # Run all tests
mvn test -pl phive-xml         # Run tests for a single module
mvn test -pl phive-api -Dtest=ValidationResultListTest  # Run a single test class
```

Requires Java 17+ and Maven 3.9+. Tested on Java 17, 21, and 25.

## Module Architecture

Six Maven modules, layered bottom-up:

- **phive-api** — Core interfaces and types: `IValidationSource`, `IValidationExecutor`, `IValidationArtefact`, `ValidationResult`, `ValidationResultList`, `ValidationExecutorSetRegistry`, `VESID`
- **phive-xml** — XML-specific validation executors: `ValidationExecutorXSD`, `ValidationExecutorSchematron`. Supports Schematron engines: XSLT, SchXslt, SchXslt2, Pure
- **phive-result** — Result serialization to JSON (`PhiveJsonHelper`) and XML (`PhiveXMLHelper`)
- **phive-ves-model** — VES (Validation Execution Set) XML data model with JAXB-generated classes (generated from `src/main/resources/schemas/`)
- **phive-ves-repo** — Repository management extensions integrating with ph-diver
- **phive-ves-engine** — VES execution engine that loads and runs VES definitions

Dependency flow: `phive-api` → `phive-xml` → `phive-result` → `phive-ves-model` → `phive-ves-engine`. `phive-ves-repo` extends `phive-ves-model`.

## Validation Execution Flow

1. Register `ValidationExecutorSet` instances (identified by `VESID`) in a `ValidationExecutorSetRegistry`
2. Create `ValidationSourceXML` from the document to validate
3. Execute via `ValidationExecutionManager` which runs each layer (XSD, then Schematron rules)
4. Collect `ValidationResultList` containing per-layer results
5. Optionally transform results via `PhiveJsonHelper` or `PhiveXMLHelper`

## Key Dependencies

- **ph-commons** (12.1.3) — Core utilities, error handling, collections
- **ph-schematron** (9.1.1) — Schematron validation engines
- **ph-diver** (4.2.0) — DVR coordinate handling and repository management
- **JAXB 4.0** — XML binding for VES model

## Conventions

- Test framework is JUnit 4 (not JUnit 5)
- Null-safety annotations from JSpecify and javax.annotation (`@NonNull`, `@Nullable`, `@Nonnegative`, `@Nonempty`)
- JAXB classes in `phive-ves-model` are auto-generated by `ph-jaxb-plugin` — do not edit files under generated source directories
- Package root: `com.helger.phive` (sub-packages: `.api`, `.xml`, `.result`, `.ves.model`, `.ves.repo`, `.ves.engine`)
